## 摘要翻译

### 英文摘要
Scientific ideation aims to propose novel solutions within a given scientific context. Existing LLM-based agentic approaches emulate human research workflows, yet inadequately model scientific reasoning, resulting in surface-level conceptual recombinations that lack technical depth and scientific grounding. To address this issue, we propose **MoRI** (**Mo**tivation-grounded **R**easoning for Scientific **I**deation), a framework that enables LLMs to explicitly learn the reasoning process from research motivations to methodologies. The base LLM is initialized via supervised fine-tuning to generate a research motivation from a given context, and is subsequently trained under a composite reinforcement learning reward that approximates scientific rigor: (1) entropy-aware information gain encourages the model to uncover and elaborate high-complexity technical details grounded in ground-truth methodologies, and (2) contrastive semantic gain constrains the reasoning trajectory to maintain conceptually aligned with scientifically valid solutions. Empirical results show that MoRI significantly outperforms strong commercial LLMs and complex agentic baselines across multiple dimensions, including novelty, technical rigor, and feasibility.

### 中文翻译
科学创新旨在给定科学背景下提出新颖的解决方案。现有的基于 LLM 的智能体方法模仿人类研究工作流，但对科学推理建模不足，导致产生缺乏技术深度和科学依据的表面层次概念重组。为了解决这个问题，我们提出了 **MoRI**（**动**机驱动的科学创**新推**理），这是一个使 LLM 能够显式学习从研究动机到方法论的推理过程的框架。基础 LLM 通过监督微调初始化，从给定上下文生成研究动机，随后在近似科学严谨性的组合强化学习奖励下训练：（1）熵感知信息增益鼓励模型揭示和阐述基于真实方法论的高复杂度技术细节，（2）对比语义增益约束推理轨迹保持与科学有效解决方案的概念一致性。实验结果表明，MoRI 在新颖性、技术严谨性和可行性等多个维度上显著优于强大的商业 LLM 和复杂的智能体基线。

### 核心要点提炼
- **研究背景**：现有 LLM 智能体框架模仿研究工作流，但缺乏对科学推理的深度建模，产生的创新想法技术深度不足
- **研究动机**：需要让 LLM 内化从研究动机到方法论的推理过程，而非依赖外部脚手架
- **核心方法**：MoRI 框架通过动机驱动的强化学习，使用组合奖励（熵感知信息增益 + 对比语义增益）优化推理
- **主要结果**：在新颖性、技术严谨性和可行性三个维度上显著优于 GPT-4o、Claude-3.5 和智能体基线
- **研究意义**：首次提出动机驱动的推理学习框架，将科学创新从表面模仿转向深层推理

## 研究背景与动机

### 领域现状
科学发现自动化正在经历范式转变。从早期的基于规则的专家系统和文献发现算法，到如今由 LLM 驱动的生成式方法。当前的科学创新方法主要采用智能体范式：

1. **智能体框架**：AI-Scientist、ResearchAgent、VirSci 等
2. **迭代工作流**：多轮研究、多智能体辩论、自主同行评审
3. **外部脚手架**：依赖人类设计的启发式规则而非增强内部推理能力

### 现有方法的局限性

**1. 表面层次的概念重组**
- 缺乏对研究动机的深度理解
- 生成的想法看似新颖但技术深度不足
- 缺乏科学依据的支撑

**2. 依赖外部脚手架**
- 复杂的迭代流水线设计
- 计算成本高昂
- 缺乏可扩展性和有效性

**3. 缺乏内在推理能力**
- 模型未内化科学推理过程
- 无法从动机到方法论进行逻辑推导
- "Bitter Lesson"：外部脚手架不如内化推理能力

### 研究动机
作者提出核心观点：**高质量的科学想法不应是上下文补全，而是动机驱动的问题解决过程**。

关键洞察：
- 科学创新需要一个推理桥梁连接研究动机和方法论
- 应该内化推理过程而非依赖外部框架
- 通过强化学习优化推理轨迹

## 研究问题

### 核心研究问题
**如何让 LLM 内化科学推理过程，从研究动机出发生成技术深度足够且科学严谨的创新想法？**

具体包括：
1. 如何建模从动机到方法论的推理过程？
2. 如何在没有确定性验证器的情况下引导强化学习？
3. 如何平衡技术深度与概念方向？

## 方法概述

### 核心思想
MoRI 的核心思想是将科学创新建模为**动机驱动的条件推理问题**：

**过程分解**：
```
研究上下文 x → 研究动机 m → 推理轨迹 z → 方法论 y
```

**关键创新**：
- 显式建模推理轨迹 z 作为动机和方法论之间的逻辑桥梁
- 通过强化学习内化推理模式
- 组合奖励机制平衡微观技术细节和宏观逻辑方向

### 方法框架

#### 整体架构

MoRI 采用两阶段框架：

**阶段1：动机制定**
- 输入：研究上下文 x（主题、背景、关键文献）
- 输出：研究动机 m（研究空白、科学原理、高层方向）
- 模型：动机制定策略 π_φ

**阶段2：推理驱动创新**
- 输入：上下文 x + 动机 m
- 输出：推理轨迹 z + 方法论 y
- 模型：推理驱动创新策略 π_θ

#### 各模块详细说明

**模块1：数据构建和初始化**

- **功能**：构建科学创新数据集并初始化模型
- **输入**：ICLR 2024-2025 论文 PDF
- **输出**：结构化数据 D = {(x_i, m_i, z_i, y*_i)}
- **处理流程**：
  1. 使用 MinerU 将 PDF 转换为 Markdown
  2. 使用 Qwen3-235B 提取研究上下文 x 和方法 y*
  3. 去符号化：移除引用标记，重写数学公式为自然语言
  4. 提取研究动机 m
  5. 后验重构：反向工程推理轨迹 z
- **关键技术**：后验重构策略、去符号化
- **数据规模**：8000 训练样本，83 测试样本（时间划分）

**模块2：监督微调初始化**

- **功能**：建立基础能力
- **训练任务**：
  1. 动机生成（x → m）
  2. 方法生成（x → (z, y)）
- **关键技术**：多任务 SFT

**模块3：动机驱动的强化学习**

- **功能**：优化推理轨迹
- **算法**：Group Relative Policy Optimization (GRPO)
- **输入**：联合输入 q = x ⊕ m
- **优化目标**：
  ```
  J(θ) = E[1/Σ|o_i| Σ min(r_{i,t} A_i, clip(r_{i,t}) A_i)]
  ```
  其中 A_i 是通过组归一化计算的优势函数
- **关键技术**：Token 级损失、Clip-Higher

**模块4：组合奖励建模**

这是 MoRI 的核心创新。奖励框架包含两个互补维度：

**4.1 熵感知信息增益 (EAIG)**

- **目标**：确保技术深度
- **选择策略**：使用熵过滤高复杂度技术细节
  - 计算每个 token 的熵
  - 选择 top 25% 高熵 token 作为掩码 M
- **评估指标**：点对信息增益
  ```
  g_t(z) = log π_θ(y*_t|x,m,z,y*_{<t}) - log π_sft(y*_t|x,m,y*_{<t})
  ```
- **最终奖励**：
  ```
  Δ_IG(z) = 1/ΣM_t Σ M_t · g_t(z)
  ```

**关键洞察**：聚焦高熵区域，奖励模型阐明复杂技术核心的能力。

**4.2 对比语义增益 (CSG)**

- **目标**：确保逻辑方向正确
- **全局意图对齐**：
  ```
  S_gen = CosSim(E(ŷ), E(y*))
  ```
- **反事实基线**：
  ```
  S_base = CosSim(E(x⊕m), E(y*))
  ```
  测量通过简单复制输入可达到的语义分数
- **对比评估**：
  ```
  Δ_sem = S_gen - S_base
  ```
  正值表示模型成功从问题空间迈向解决方案空间

**关键洞察**：奖励真正的语义跃升而非重新陈述背景信息。

**4.3 奖励协同**

- **EAIG**：微观验证器（技术深度）
- **CSG**：宏观向导（逻辑方向）
- **协同效应**：防止模型产生流利但空洞的陈词滥调或产生脱离熵指标的术语

**最终组合奖励**：
```
R_total = α(z) · 1[valid] · (w_1 f_step(Δ_IG) + w_2 f_step(Δ_sem))
```

其中：
- **长度锚定 α(z)**：惩罚过短推理，防止认知捷径
- **格式约束 1[valid]**：防止奖励黑客（空推理、格式泄露）

### 关键设计原则

**1. 动机驱动的推理**
- 显式建模从动机到方法论的过程
- 强制模型进行动机驱动的问题解决

**2. 双粒度奖励**
- 微观：技术细节深度（EAIG）
- 宏观：逻辑方向对齐（CSG）

**3. 鲁棒性保障**
- 长度锚定防止推理坍塌
- 格式检查防止奖励黑客

## 实验结果

### 实验目标
验证 MoRI 在科学创新任务上的有效性：
1. 相比商业模型和智能体框架的性能
2. 组合奖励组件的贡献
3. RL 是否成功内化深度推理模式

### 实验设置

#### 基线方法
1. **商业模型**：GPT-4o, Claude-3.5-Sonnet
2. **智能体框架**：
   - AI-Scientist-V2
   - ResearchAgent
   - VirSci
3. **内部基线**：Full-SFT

#### 评估指标
1. **Novelty（新颖性）**：想法的创新程度
2. **Technical Rigor（技术严谨性）**：方法的技术深度和严谨性
3. **Feasibility（可行性）**：实际实施的可行性
4. **评估方法**：检索增强的 LLM judge (Gemini-2.5-Pro) + 人工验证

#### 实验环境
- **基础模型**：DeepSeek-R1-Distilled-Qwen-14B
- **训练框架**：GRPO
- **最优配置**：w_s=0.7, w_e=0.3, Top-25% 熵掩码, 400 训练步

> 注：**粗体**表示最优结果

#### 结果分析

**关键发现**：

1. **MoRI 全面领先**
   - 平均分达到 3.19，接近 Ground Truth (3.58)
   - 相比 Full-SFT 提升 6.7%
   - 相比最强智能体基线提升超过 17%

2. **技术严谨性显著提升**
   - MoRI 在技术严谨性上达到 3.16，超过 Claude-3.5 (3.07)
   - 相比 GPT-4o 提升 13.7%
   - 验证了 EAIG 奖励的有效性

3. **可行性优势明显**
   - MoRI 在可行性上达到 3.11，显著高于 Claude-3.5 (2.82)
   - 提升 10.3%，证明生成的方法更具实际价值

4. **商业模型的局限性**
   - Claude-3.5 在新颖性上表现优异 (3.39)，但可行性不足
   - 验证了商业模型擅长概念创新但缺乏实用性的观察

### 消融实验

#### 奖励组成消融

| w_s | w_e | 描述 | 新颖性 | 技术严谨性 | 可行性 | 平均分 |
|-----|-----|------|--------|------------|--------|--------|
| 0.0 | 1.0 | EAIG Only | 2.68 | 2.22 | 2.63 | 2.51 |
| 1.0 | 0.0 | CSG Only | 3.16 | 2.93 | 3.06 | 3.05 |
| 0.5 | 0.5 | Balanced | 3.32 | 2.96 | 2.98 | 3.09 |
| **0.7** | **0.3** | **Optimal** | **3.34** | **3.04** | **3.07** | **3.15** |

**关键发现**：
- **EAIG 单独使用导致失败**：平均分仅 2.51，训练不稳定
- **CSG 单独使用次优**：平均分 3.05，推理长度快速增长但缺乏约束
- **最优配置 (w_s=0.7, w_e=0.3)**：强语义方向 + 适度精度约束

**训练动态分析**：
- EAIG 单独：奖励坍塌 + 长度爆炸 → 生成不连贯内容
- CSG 单独：相对稳定但推理长度快速增长
- 最优配置：稳定的推理长度增长，避免坍塌

#### 熵掩码消融

| 熵掩码 | 新颖性 | 技术严谨性 | 可行性 | 平均分 | 相对提升 |
|--------|--------|------------|--------|--------|----------|
| Top 50% | 3.16 | 2.78 | **3.00** | 2.98 | -- |
| **Top 25%** | **3.32** | **2.96** | 2.98 | **3.09** | **+3.7%** |

**关键发现**：
- 更严格的熵过滤（Top 25%）带来 3.7% 提升
- 技术严谨性提升 0.18 分，新颖性提升 0.16 分
- 50% 掩码引入功能性词汇噪声，干扰优化

#### 长度锚定消融

| 配置 | 长度锚定 | 新颖性 | 技术严谨性 | 可行性 | 平均分 | 相对提升 |
|------|----------|--------|------------|--------|--------|----------|
| s7-e3 | ✗ | 3.22 | 2.94 | 3.00 | 3.05 | -- |
| s7-e3 | ✓ | **3.34** | **3.04** | **3.07** | **3.15** | **+3.2%** |
| s5-e5 | ✗ | 2.96 | 2.76 | 2.95 | 2.89 | -- |
| s5-e5 | ✓ | **3.32** | **2.96** | 2.98 | **3.09** | **+6.9%** |

**关键发现**：
- 移除长度锚定导致性能全面下降
- s5-e5 配置中，新颖性大幅下降 (3.32 → 2.96)
- 训练动态显示：无锚定时推理长度急剧衰减（认知捷径）

## 深度分析

### 研究价值评估

#### 理论贡献
- **贡献1**：动机驱动的科学创新框架
  - 创新点：首次将科学创新建模为动机驱动的条件推理
  - 学术价值：为 AI 辅助科学研究提供新范式
  - 影响范围：所有需要创新思维的科学领域

- **贡献2**：组合奖励机制
  - 创新点：双粒度奖励（微观技术深度 + 宏观逻辑方向）
  - 学术价值：解决开放性任务的强化学习引导问题
  - 影响范围：其他无确定性验证器的领域

- **贡献3**：后验重构方法
  - 创新点：反向工程推理轨迹填补数据空白
  - 学术价值：为其他推理任务提供数据构建思路
  - 影响范围：需要推理链的训练场景

#### 实际应用价值
- **应用场景1**：研究想法生成
  - 适用性：科研人员在早期探索阶段使用
  - 优势：提供技术深度足够的创新想法
  - 潜在影响：加速研究迭代、拓宽研究视野

- **应用场景2**：研究方向规划
  - 适用性：确定研究空白和潜在方向
  - 优势：系统化的动机分析和方向推导
  - 潜在影响：减少研究方向选择的盲目性

- **应用场景3**：教育和培训
  - 适用性：帮助学生理解科研思维过程
  - 优势：展示从动机到方法论的推理链
  - 潜在影响：降低科研入门门槛

#### 领域影响
- **短期影响**：提供高质量的科学想法生成工具
- **中期影响**：改变科研辅助工具的设计思路（从外部脚手架到内化推理）
- **长期影响**：推动 AI 成为真正的科研合作伙伴
- **潜在变革**：科学发现过程的范式转变

### 方法优势详解

#### 优势1：内化推理能力
- **描述**：通过 RL 内化推理模式，而非依赖外部规则
- **技术基础**：动机驱动的条件推理建模
- **实验验证**：显著优于复杂的智能体框架（+17%）
- **对比分析**：验证了 "Bitter Lesson" 的观点

#### 优势2：技术深度保证
- **描述**：EAIG 确保生成想法的技术深度
- **技术基础**：熵感知的信息增益奖励
- **实验验证**：技术严谨性达到 3.16，超过 Claude-3.5
- **对比分析**：解决了商业模型"概念新颖但技术浅薄"的问题

#### 优势3：训练稳定性
- **描述**：组合奖励机制避免训练坍塌
- **技术基础**：EAIG + CSG 协同 + 长度锚定
- **实验验证**：最优配置保持稳定的训练动态
- **对比分析**：单一 EAIG 导致奖励坍塌，组合机制提供鲁棒性

### 局限性分析

#### 局限1：领域局限
- **描述**：仅在计算机科学（机器学习）领域验证
- **表现**：未测试生物学、物理学等其他学科
- **原因**：数据集来源局限于 ICLR
- **影响**：泛化性有待验证
- **可能的解决方案**：扩展到其他领域的论文数据

#### 局限2：评估主观性
- **描述**：新颖性和可行性难以客观评估
- **表现**：依赖 LLM judge 和人工评估
- **原因**：科学创新缺乏确定性标准
- **影响**：评估结果可能存在偏差
- **可能的解决方案**：
  - 更大规模的同行评审
  - 实际实验验证可行性

#### 局限3：计算成本
- **描述**：RL 训练需要大量计算资源
- **表现**：400 训练步需要显著 GPU 时间
- **原因**：GRPO 需要采样多个输出计算优势函数
- **影响**：限制了大规模应用
- **可能的解决方案**：更高效的 RL 算法、知识蒸馏

#### 局限4：数据依赖
- **描述**：需要高质量的研究动机和推理轨迹标注
- **表现**：后验重构可能不完全反映真实推理过程
- **原因**：科学论文通常不显式展示推理链
- **影响**：数据质量影响模型性能
- **可能的解决方案**：专家标注推理链
