## 摘要翻译

### 英文摘要
Large language models (LLMs) exhibit remarkable reasoning capabilities through reinforcement learning (RL) with verifiable rewards. However, existing label-free RL methods suffer from a critical limitation: during group-based verification, models tend to maximize self-consistency by converging to identical outputs, leading to output diversity collapse and systematic error reinforcement. We identify this phenomenon as the "consensus trap" and propose CoVerRL, a novel generator-verifier co-evolution framework that breaks this trap through dual-role alternating training. Our key insight is that a single model can alternately扮演 generator and verifier roles, with majority voting providing pseudo-labels to train the verifier, which then filters consensus errors to create a virtuous cycle. We further introduce Answer-Anchored GRPO, a modified grouping strategy that anchors verification to参考答案 for more reliable reward signals. Experiments on Qwen and Llama series models show that CoVerRL improves self-verification accuracy from 55% to 85%+, outperforming TTRL by 4.7-5.9% on mathematical reasoning benchmarks.

### 中文翻译
大语言模型（LLM）通过可验证奖励的强化学习（RL）展现出卓越的推理能力。然而，现有的无标签 RL 方法存在一个关键局限性：在基于群体的验证过程中，模型倾向于通过收敛到相同输出来最大化自一致性，导致输出多样性崩溃和系统性错误强化。我们将这一现象识别为"共识陷阱"，并提出 CoVerRL，这是一种新颖的生成器 - 验证器协同进化框架，通过双角色交替训练打破这一陷阱。我们的核心洞见是，单一模型可以交替扮演生成器和验证器角色，通过多数投票提供伪标签来训练验证器，然后验证器过滤共识错误以形成良性循环。我们进一步引入了 Answer-Anchored GRPO，一种改进的分组策略，将验证锚定到参考答案以获得更可靠的奖励信号。在 Qwen 和 Llama 系列模型上的实验表明，CoVerRL 将自验证准确率从 55% 提升至 85% 以上，在数学推理基准上超越 TTRL 4.7-5.9%。

### 核心要点提炼
- **研究背景**：无标签 RL 推理方法中，群体验证导致输出多样性崩溃
- **研究动机**：需要打破共识陷阱，避免模型自信地强化系统性错误
- **核心方法**：生成器 - 验证器协同进化 + Answer-Anchored GRPO
- **主要结果**：验证准确率 55%→85%+，数学推理提升 4.7-5.9%
- **研究意义**：为无标签强化学习推理提供了新的训练范式

## 研究背景与动机

### 领域现状
通过强化学习提升 LLM 推理能力已成为研究热点，主要方法包括：
- **RLHF**：使用人类反馈进行强化学习
- **TTRL**：通过测试时扩展和自一致性验证提升推理
- **Label-free RL**：无需标注数据，通过自验证进行训练

### 现有方法的局限性

**共识陷阱（Consensus Trap）**：
1. **多样性坍塌**：模型为了最大化自一致性，倾向于生成相同输出
2. **系统性错误强化**：当多数输出都错误时，模型会自信地强化错误
3. **验证器退化**：验证器无法区分高质量和低质量输出

**根本问题**：
- 生成器和验证器独立训练，缺乏协同进化
- 验证信号依赖于不可靠的多数投票
- 没有机制防止群体性错误

### 研究动机
受人类认知启发：
- 人类在推理时会交替扮演"生成者"和"审查者"角色
- 生成阶段追求创意和多样性，验证阶段追求严谨和准确
- 两种角色相互促进，形成良性循环

## 研究问题

### 核心研究问题
**如何在无标签设置下设计生成器 - 验证器协同进化机制，打破共识陷阱，实现可靠的推理能力提升？**

子问题：
1. 如何让单一模型有效扮演双重角色？
2. 如何在缺乏真实标签时提供可靠的验证信号？
3. 如何防止多数投票导致的系统性错误强化？

## 方法概述

### 核心思想
CoVerRL 的核心思想是**双角色协同进化**：

```
单模型 M
   ↓
┌─────────────────────────────────┐
│ 交替训练：                       │
│ - 生成器模式：生成多样化候选答案 │
│ - 验证器模式：评估答案正确性     │
│ - 多数投票：提供伪标签           │
│ - 过滤错误：打破共识陷阱         │
└─────────────────────────────────┘
   ↓
良性循环：验证器质量提升 → 过滤更准确 → 训练数据更优 → 生成器质量提升
```

### 方法框架

#### 整体架构

![CoVerRL 框架图|800](images/method_final_yc.pdf)

> 图 1：CoVerRL 整体框架。单一模型交替扮演生成器和验证器角色，通过 Answer-Anchored GRPO 进行分组验证，形成协同进化循环。

**核心组件**：

1. **生成器模式（Generator Mode）**
   - 输入问题，生成 N 个候选答案
   - 使用多样性采样策略（temperature > 1.0）
   - 输出：{A₁, A₂, ..., Aₙ}

2. **验证器模式（Verifier Mode）**
   - 输入问题和候选答案，输出正确性评分
   - 使用 Answer-Anchored GRPO 进行分组
   - 输出：{s₁, s₂, ..., sₙ}

3. **Answer-Anchored GRPO**
   - 将答案按参考答案分组
   - 同组内答案共享奖励信号
   - 减少方差，提高训练稳定性

4. **协同进化循环**
   - 生成器产生多样化候选
   - 验证器筛选高质量答案
   - 筛选后的数据训练生成器
   - 更好的生成器产生更优质候选

#### 各模块详细说明

**模块 1：共识陷阱识别**

![共识陷阱示意图|600](images/intro_small.pdf)

> 图 2：共识陷阱示意图。传统方法中，模型为了最大化自一致性，导致输出多样性坍塌，系统性地强化错误。

**问题形式化**：
```
传统方法：
- 生成 N 个答案：{A₁, A₂, ..., Aₙ} ~ π_θ(·|q)
- 多数投票：A_majority = argmax_A count(A)
- 奖励：R = 1 if A_majority is correct else 0

问题：
- 当多数答案错误时，模型接收到错误奖励
- 模型学习到"自信地犯错"
```

**模块 2：双角色协同进化**

**训练流程**：
```
迭代 t:
1. 生成器阶段：
   - 采样答案：{A₁, ..., Aₙ} ~ π_θ_gen(·|q)
   - 多样性约束：temperature = τ_gen > 1.0

2. 验证器阶段：
   - 评估答案：{s₁, ..., sₙ} = π_θ_ver(q, A₁, ..., Aₙ)
   - 一致性检查：flag = detect_consensus_trap({s_i})

3. 伪标签生成：
   - 如果未陷入陷阱：多数投票提供标签
   - 如果陷入陷阱：使用参考答案锚定

4. 参数更新：
   - 生成器损失：L_gen = -E[log π_θ_gen(A_correct|q)]
   - 验证器损失：L_ver = -E[log π_θ_ver(s_correct|q, A)]
   - 共享参数：θ_gen = θ_ver = θ（部分共享）
```

**模块 3：Answer-Anchored GRPO**

**传统 GRPO 的问题**：
- 每组答案独立计算奖励，方差大
- 相似答案可能得到不同奖励

**Answer-Anchored 改进**：
```
1. 答案分组：
   - 基于语义相似度聚类：{C₁, C₂, ..., Cₖ}
   - 每组答案共享同一参考答案

2. 组内奖励计算：
   - 对于组 C_j 中的所有答案，使用同一参考标准
   - R(A_i) = similarity(A_i, anchor_j) × correctness

3. 优势函数：
   - A_i = R(A_i) - mean(R(C_j))
   - 减少组内方差，提高训练稳定性
```

**模块 4：共识陷阱检测与恢复**

**检测机制**：
```
输入：验证器评分 {s₁, s₂, ..., sₙ}

指标：
1. 一致性分数：C = std({s_i}) / mean({s_i})
2. 多样性分数：D = unique_answers / N
3. 置信度分数：Conf = max(s_i) - mean(s_i)

陷阱判定：
- 如果 C < τ_c (低方差) 且 D < τ_d (低多样性)：
  → 检测到共识陷阱
```

**恢复策略**：
```
1. 提高生成温度：τ_gen ← τ_gen × 1.2
2. 增加验证器正则化：鼓励区分不同答案
3. 使用参考答案锚定：绕过多数投票
```

### 算法伪代码

```
算法 1: CoVerRL 训练流程

输入：问题集 Q，模型π_θ，迭代次数 T
输出：训练好的模型π_θ*

for t = 1 to T do
    采样问题 q ~ Q

    /* 1. 生成器阶段 */
    设置 temperature = τ_gen
    生成答案 {A₁, A₂, ..., Aₙ} ~ π_θ(·|q)

    /* 2. 验证器阶段 */
    设置 temperature = τ_ver
    计算验证评分 {s₁, s₂, ..., sₙ} = π_θ_ver(q, {A_i})

    /* 3. 共识陷阱检测 */
    if detect_consensus_trap({s_i}) then
        使用参考答案锚定策略
        获取伪标签 {y₁, y₂, ..., yₙ}
    else
        使用多数投票
        获取伪标签 {y₁, y₂, ..., yₙ}
    end if

    /* 4. Answer-Anchored GRPO */
    对答案进行语义聚类：{C₁, C₂, ..., Cₖ}
    for each cluster C_j do
        计算组内奖励：R(A_i) for A_i ∈ C_j
        计算优势函数：A_i = R(A_i) - mean(R(C_j))
    end for

    /* 5. 参数更新 */
    计算生成器损失：L_gen = -Σ log π_θ(A_i|q) × A_i
    计算验证器损失：L_ver = -Σ log π_θ(s_i|q, A_i) × (s_i - y_i)²
    更新参数：θ ← θ - α∇(L_gen + λL_ver)

end for

返回 π_θ*
```

- **基础模型**：Qwen2.5-7B, Qwen2.5-14B, Llama-3.1-8B

- **原因**：缺乏明确的验证信号
- **影响**：应用场景受限
- **解决方案**：探索弱监督或人类反馈

#### 局限 2：计算成本较高
- **描述**：每次迭代需要生成 N 个答案并验证
- **表现**：训练时间是标准 SFT 的 N 倍
- **原因**：群体验证需要多次前向传播
- **影响**：资源需求高
- **解决方案**：探索更高效的采样策略

#### 局限 3：超参数敏感性
- **描述**：温度参数、分组数等超参数需要调优
- **表现**：不同任务需要不同的超参数配置
- **原因**：任务难度和领域差异
- **影响**：增加了使用门槛
- **解决方案**：开发自适应超参数调整

### 场景分析

#### 适用场景
- **数学推理**：有明确答案，可自动验证
- **代码生成**：可通过测试用例验证正确性
- **科学问答**：有标准答案的知识性问题
- **逻辑推理**：可形式化验证的推理任务

#### 不适用场景
- **创意写作**：缺乏明确验证标准
- **开放域对话**：答案多样性高，难以验证
- **主观评价任务**：如艺术评论、情感分析
